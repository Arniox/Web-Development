What Special Features are there: Troubles and issues I had: What would I try to do better next time: What have I learnt along the way: What refrences/sources where used:
Alot of my special features are more style based rather than functionality, but I have a few functional special features as well.
  • My biggest special feature is that I'm using bootstrap CSS styling for everything on my whole website. From buttons, to radio buttons to text fields and so on. This is generally for a much cleaner nicer looking website with a more modern approach
  • In my post status form, I put a lot of work into making the validation occure INSIDE the post status form and then the validated data being sent to the post status process where it's further validated by checking to see if it already exists in the database and so on.
    This means I can get really nice dynamic CSS styling inside my post status form with PHP validation. I wanted to make sure this worked because I personally think it looks way better than a seperate error message
  • I also wanted a really nice layout to the search status form so I went with a card like layout which looks very nice
  • The navigation bar is another nice feature I decided to add
I didn't have many issues during the first few parts of the website. Just simple stuff with styling. These are all issues I faced after that starting period:
  • My biggest issue of all had to do with how bad PHP is. For some damn reason if I defined a pattern such as my status code pattern outside of the check status code function; the function would not recognise the regular expression.
    I had to define the regular expression inside each check function and then it finally worked. I spent over 2 days on this issue and was about to give up and wait to ask the teacher but I eventually figured it out through testing and sheer luck.
  • One of my other really big issues was the SQL intergration. I ended up with some simple bugs but they kept me busy and trying to fix them for a long time. One of the bugs I had was that I was generating a query, but I wasn't running it and getting any results -_-
    Things like this slowed my progression down a lot because I often had to wait till the labs to ask some friends what I was doing wrong
  • Another big issue I had was posting or linking to another php document from the current php document without using the html form. I was trying to just use php but in the end I abonded the idea and decided to do a an automatic submission once the values where all validated using javascript. Once all the values are submitted, javascript is inserted into the page which then automatical re-submits the document. On the user side they don't notice anything.
  • Another big issue I had; mostly occuring on the search status process php document, was scaling with bootstrap. Bootstrap is kinda hard to use since there is so many different classes you can use to style something. I tried to keep a consistent styling theme going across all my documents but doing that was really hard due to each page having slightly different styling.
  • Another really big issue I had which was entirely not my fault, was that for some reason, whenever I would work from home and use remote desktop to drag and drop files from my home computer into WinSCP it would sometimes not update at all. I would literally be able to delete all the files for my entire assignement off of WinSCP and it would somehow still be able to work in my browser. In order to fix this I had to completely log off and restart the entire remote connection and then delete the entire assignement folder off of the remote desktop and WinSCP and then recopy it from my comuter to the remote desktop and WinSCP. It was extreemely tedious and insanely annoying but I finally got it working.
  • I think one of the biggest things I would do next time is not spend so much time on styling. I effectively waited until the last week before working on the majority of the functionality because I wanted the website to look really nice.
  • I would also probably keep it simple and not try to do multiple submits inside 1 php form like I did in the post status form.
